# Contributing to Cattlelog

Thank you for your interest in contributing to Cattlelog! This project welcomes contributions from everyone.

## Students

If you're here for an assignment:

1. Read the [assignment instructions](docs/INSTRUCTIONS.md)
2. Follow the [fork workflow](docs/guides/fork-workflow.md)
3. Use the [checklist](docs/CHECKLIST.md) before submitting

**Need help?** See [troubleshooting](docs/guides/troubleshooting.md) or open an issue using the "Help with creature" template.

## Other contributors

We welcome bug fixes, documentation improvements, and new features but we ask that new cow contributions **only** come from the repo maintainer's students.

### Quick start

```bash
git clone https://github.com/YOUR-USERNAME/cattlelog.git
cd cattlelog
npm install
npm run dev
```

### Before you submit

- Run `npm test` to ensure tests pass
- Follow existing code style
- Keep changes focused and minimal

### Types of contributions

| Type          | Notes                                        |
| ------------- | -------------------------------------------- |
| Bug fixes     | Open an issue first to discuss               |
| Documentation | Improvements always welcome                  |
| New creatures | Follow the standard creature format          |
| Features      | Open an issue to discuss before implementing |

## Content guidelines

All contributions are subject to review. Maintainers reserve the right to reject any submission for any reason without explanation. This includes but is not limited to:

- ASCII art or text that may be inappropriate for an educational setting
- Content that does not align with project goals
- Submissions that do not follow contribution guidelines

## Code of conduct

This project follows the [Contributor Covenant](CODE_OF_CONDUCT.md). By participating, you agree to uphold this code.
